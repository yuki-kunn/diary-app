# 🌿 ひだまり日記

毎日の出来事や写真を保存して、あとからアーカイブとして見返せる日記帳PWAアプリ。

## 機能

- **パスコードロック(ログイン)** — 4桁パスコードでアプリを保護(設定でオン/オフ)
- **カレンダービュー** — 月表示。日記のある日はドット、写真のある日はサムネイル表示。日付タップで日記作成/編集
- **日記作成** — タイトル・本文・気分(絵文字)・写真(複数枚、自動で最大1600pxに圧縮)
- **ギャラリー** — すべての写真を新しい順に一覧。タップで拡大 → 日記へジャンプ
- **設定** — 通知のオン/オフ、パスコード変更、データのエクスポート/インポート、使用容量表示
- **PWA** — ホーム画面に追加してオフラインでも利用可能
- **リマインド通知** — 毎日22時(JST)を過ぎて日記が未登録の場合にプッシュ通知

## 技術構成

| 層 | 技術 |
|---|---|
| フロントエンド | Vanilla JS SPA + Service Worker + IndexedDB(日記・写真は端末内に保存) |
| バックエンド | Node.js + Express(静的配信・プッシュ購読管理のみ。日記データはサーバーに送信されない) |
| 通知 | Web Push (VAPID) + node-cron(22:00 JST に未登録者へ送信) |

## 起動方法

```bash
cd ~/diary-app
npm install
npm start
# → http://localhost:3000
```

VAPID鍵は初回起動時に自動生成され `data/vapid.json` に保存されます。

## スマホでの通知(重要)

プッシュ通知には **HTTPS(またはlocalhost)** が必須です。スマホから使う場合:

1. サーバーをHTTPSで公開する(例: Cloudflare Tunnel、ngrok、VPS + Let's Encrypt など)
   ```bash
   # 例: ngrok
   ngrok http 3000
   ```
2. スマホのブラウザでURLを開く
3. **iPhone (iOS 16.4+)**: 共有メニュー →「ホーム画面に追加」→ 追加したアイコンから起動 → 設定画面で通知をオン
   ※ iOSではホーム画面に追加したPWAからのみ通知が使えます
4. **Android**: Chromeで開いて「アプリをインストール」→ 設定画面で通知をオン

22時のリマインドは**サーバーが起動している間**送信されます。常時通知したい場合はサーバーを常駐させてください(例: `pm2 start server.js`)。

## データについて

- 日記・写真はすべて**端末のIndexedDB内**に保存され、サーバーには送信されません
- サーバーが保持するのは通知用の購読情報と「最後に日記を書いた日付」のみ
- 機種変更時は 設定 → エクスポート でJSONを書き出し、新端末でインポートしてください
